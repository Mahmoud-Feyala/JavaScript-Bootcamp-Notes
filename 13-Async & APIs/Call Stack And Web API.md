# JavaScript - Call Stack And Web API 

عشان نفهم الجافاسكريبت بتتعامل إزاي مع العمليات اللي بتاخد وقت (اللامتزامنة)، لازم الأول نفهم الآلية اللي بتدير بيها تنفيذ الكود سطر بسطر.

---

## 1. The Call Stack 

الـ `Call Stack` هو المكان أو "السجل" اللي محرك الجافاسكريبت بيستخدمه عشان يتتبع هو بينفذ إيه دلوقتي، وإيه الدالة اللي المفروض تتنفذ بعدها.

- **طريقة العمل**: بيشتغل بمبدأ `LIFO` (Last In, First Out) يعني "الأخير في الدخول هو الأول في الخروج". (زي طابور الأطباق فوق بعضها، لازم تسحب الطبق اللي فوق الأول عشان توصل للي تحته).
- **إضافة دالة**: لما بتستدعي دالة، بتتحط في قمة الـ Stack.
- **إزالة دالة**: أول ما الدالة تخلص تنفيذ وتعمل `return` ، بتتشال من الـ Stack، والمترجم بيكمل من النقطة اللي وقف عندها.
- **التزامن**: الـ Stack ده متزامن (`Synchronous`)، يعني بينفذ حاجة واحدة بس في نفس الوقت ومبيعرفش يوازي بين عمليتين.

---

## 2. Web API 

بما إن الجافاسكريبت `Single-Threaded` (طابور واحد)، لو فيه دالة هتاخد وقت طويل (زي جلب بيانات من سيرفر أو تايمر `setTimeout`)، الـ Call Stack هيقف والصفحة هتهنج. 

هنا بييجي دور المتصفح! الـ Call Stack أول ما بيلمح دالة تابعة للـ `Web API` (يعني دوال المتصفح هو اللي بيوفرها مش أصلية في لغة الجافاسكريبت نفسها)، بيرميها فوراً للمتصفح عشان يتعامل معاها في الخلفية ويفضي نفسه لباقي الكود السريع.

---

## 3. Practical Example 

تعالى نحلل الكود بتاعك ونشوف مين هيتطبع الأول وليه:
```javascript
// 1. المتصفح بياخد التايمر ده يرميه في الـ Web API ويبدأ يعد في الخلفية
// (حتى لو الوقت 0، هيترمى برضه بره الـ Call Stack)
setTimeout(() => {
  console.log("Web API");
}, 0);

function one() { console.log("One"); }
function two() { one(); console.log("Two"); }
function three() { two(); console.log("Three"); }

// 2. بداية تنفيذ الكود الفعلي هنا
three();
```

**إيه اللي بيحصل في الـ Call Stack؟**
1. دالة `three()` بتدخل الـ Stack.
2. دالة `three()` بتستدعي دالة `two()` ، فتدخل `two()` فوقها.
3. دالة `two()` بتستدعي دالة `one()` ، فتدخل `one()` في القمة.
4. دالة `one()` بتطبع `"One"` في الكونسول وتتشال من الـ Stack.
5. المترجم يرجع لدالة `two()` يطبع `"Two"` ويشيلها.
6. المترجم يرجع لدالة `three()` يطبع `"Three"` ويشيلها.
7. الـ Stack بقى فاضي! هنا بس المترجم يبص يشوف لو فيه حاجة خلصت في الـ Web API (اللي هي الـ `setTimeout`)، فيدخلها الـ Stack وتطبع `"Web API"`.

**النتيجة النهائية في الكونسول:**
```text
One
Two
Three
Web API
```

# JavaScript - Event Loop & Callback Queue 

دلوقتي إحنا عرفنا إن الجافاسكريبت بتستخدم الـ `Call Stack` لتنفيذ الأوامر، والـ `Web API` للعمليات اللي بتاخد وقت. السؤال هنا: لما الـ `Web API` بتخلص شغلها، إزاي بترجع الكود تاني للجافاسكريبت عشان يتنفذ؟ السر كله في الـ `Callback Queue` والـ `Event Loop`.



---

## 1. The Full Story 

دي الخطوات اللي بتحصل بالضبط من أول ما الكود بيشتغل لحد ما يخلص:

1. **المسار الواحد** (`Single Thread`): الجافاسكريبت بتنفذ العمليات في مسار واحد باستخدام الـ `Call Stack` ، ولما الدالة بتخلص بتتشال منه.
2. **العمليات اللامتزامنة** (`Asynchronous`): لو استدعينا دالة زي `setTimeout` ، الـ Stack بيبعتها فوراً للـ `Browser API`.
3. **دور المتصفح** (`Browser API`): المتصفح بيشتغل كأنه "مسار تاني" (`Second Thread`)، بيبدأ يعد الوقت أو يجيب البيانات في الخلفية.
4. **طابور الانتظار** (`Callback Queue`): لما المتصفح بيخلص، مش بيرمي النتيجة في وش الجافاسكريبت فجأة! هو بيبعت الدالة (الـ `Callback`) تقف في طابور محترم اسمه الـ `Callback Queue`. الطابور ده بيتبع نظام `FIFO` (الأول في الدخول هو الأول في الخروج).
5. **حارس المرور** (`Event Loop`): الـ `Event Loop` ده وظيفته الوحيدة إنه يراقب الـ `Call Stack`. بيفضل مستنيه لحد ما يفضى تماماً من كل الكود المتزامن، وأول ما يفضى، يسحب أول دالة من طابور الانتظار ويدخلها الـ Stack عشان تتنفذ.

---

## 2. Execution Order 

عشان نفهم دور الـ `Event Loop` ، شوف الكود ده:
```javascript
console.log("One"); // 1. هتدخل الـ Stack وتتنفذ فوراً

// 2. هتروح للـ Web API، وبعدين تقف في أول الـ Callback Queue
setTimeout(() => {
  console.log("Three");
}, 0);

// 3. هتروح للـ Web API، وبعدين تقف في الـ Callback Queue ورا اللي قبلها
setTimeout(() => {
  console.log("Four");
}, 0);

console.log("Two"); // 4. هتدخل الـ Stack وتتنفذ فوراً

// النتيجة هتكون: One -> Two -> Three -> Four
// الـ Event Loop مش هيدخل Three و Four غير لما يتأكد إن One و Two خلصوا تماماً والـ Stack بقى فاضي.
```

---

## 3. Scope And The Event Loop 

التريكة دي بتيجي كتير في الانترفيوهات. إيه اللي هيحصل للمتغير `myVar` هنا؟

```javascript
setTimeout(() => {
  console.log(myVar); // هيطبع 200 مش undefined!
}, 0);

let myVar = 100;
myVar += 100;
```

**السبب:**
دالة الـ `setTimeout` راحت للـ `Web API` وبعدين وقفت في الـ `Callback Queue`. خلال الوقت ده، الـ `Call Stack` كان بيكمل شغله العادي، فقام بتعريف المتغير `myVar` بـ 100، وبعدين زوّد عليه 100 كمان (بقى 200). 
ولما الـ Stack فِضِي، الـ `Event Loop` جاب الدالة من الطابور عشان تتنفذ، فالدالة دورت على `myVar` في الذاكرة لقته اتحدث وبقى 200، فطبعته!
