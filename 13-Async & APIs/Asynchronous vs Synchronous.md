# JavaScript - Asynchronous vs Synchronous Programming

عشان نقدر نتعامل مع الـ `API` ونجيب بيانات من السيرفر، لازم الأول نفهم إزاي الجافاسكريبت بتنفذ الأوامر. هل بتشغل الأوامر كلها مع بعض ولا سطر سطر؟

---
## 1. Synchronous Programming (متزامن)

الوضع الافتراضي للجافاسكريبت إنها متزامنة (`Synchronous`). يعني الكود بيتنفذ بالترتيب، سطر ورا سطر. كل عملية **لازم تستنى** العملية اللي قبلها تخلص عشان تبدأ.

- **مثال من الحياة**: طابور البنك. الموظف مش هيقدر يخدم العميل التاني غير لما يخلص العميل الأول بالكامل.
- **مثال برمجي**: دالة `window.alert()`. أول ما بتشتغل، بتوقف تنفيذ أي كود بعدها لحد ما اليوزر يدوس `OK`.
```javascript
// الكود المتزامن (Synchronous)
console.log("1");
console.log("2");
window.alert("Operation"); // الكود هيقف هنا تماماً ومش هيكمل
console.log("3");          // السطر ده مش هيتطبع غير لما تقفل الـ Alert
```

---

## 2. Asynchronous Programming (غير متزامن)

في البرمجة اللامتزامنة (`Asynchronous`)، العمليات اللي بتاخد وقت (زي جلب بيانات من السيرفر أو التايمر) بتشتغل في الخلفية، وباقي الكود بيكمل شغل عادي جداً من غير ما يستناها. ولما العملية الطويلة تخلص، بترجع نتيجتها.

- **مثال من الحياة**: إنت طلبت أكل في مطعم. الجرسون مش بيقف جنبك لحد ما الأكل يجهز ويوقف المطعم كله! هو بياخد طلبك يوديه المطبخ، ويروح يخدم ناس تانية، ولما أكلك يجهز بيجيبهولك.
- **مثال فيسبوك**: لما بتفتح الفيسبوك، الموقع بيحمل بسرعة، والبوستات أو الصور التقيلة بتفضل تحمل في الخلفية من غير ما تمنعك إنك تعمل سكرول أو تدوس لايك.
- 
```javascript
// الكود اللامتزامن (Asynchronous)
console.log("1");
console.log("2");

// العملية دي هتاخد 3 ثواني، المتصفح هيرميها في الخلفية ويكمل الكود
setTimeout(() => console.log("Operation"), 3000);

console.log("3"); 
// النتيجة هتكون: 1 -> 2 -> 3 -> وبعد 3 ثواني يطبع Operation
```

## 3. JavaScript Threads 

ده ملخص البحث عن الـ `Threads` (مسارات التنفيذ في المعالج):

- **لغات أحادية المسار** (`Single-Threaded`): الجافاسكريبت بتشتغل في مسار واحد بس (طابور واحد). يعني مينفعش تنفذ عمليتين مع بعض في نفس اللحظة بالظبط. عشان كده بتعتمد على المتصفح (Web APIs) إنه ياخد منها الشغل التقيل (زي الـ `setTimeout` أو الـ `AJAX`) يخلصه في الخلفية ويرجعلها النتيجة، عشان الطابور الأساسي ميهنجش.
- **لغات متعددة المسارات** (`Multi-Threaded`): لغات تانية زي `Java` أو `C++` تقدر تفتح أكتر من مسار في المعالج، وتنفذ أكتر من عملية معقدة في نفس اللحظة الحقيقية.
